## Airレジ

### 基本情報

| 項目 | 内容 |
|-----|------|
| **開発元** | 株式会社リクルート |
| **対応度** | ✅ 完全対応 |
| **推奨エクスポート** | 売上日報 |
| **注意点** | 店舗IDの追加が必要 |

---

### CSVエクスポート手順

#### ステップ1: Airレジアプリにログイン

```
1. iPadでAirレジアプリを開く
2. 管理者アカウントでログイン
3. メニューボタン（≡）をタップ
```

#### ステップ2: 売上日報の表示

```
1. メニューから「集計」をタップ
2. 「売上日報」を選択
3. 期間を指定
   - 開始日: エクスポートしたい最初の日
   - 終了日: エクスポートしたい最後の日
4. 「表示」をタップ
```

#### ステップ3: CSVエクスポート

```
1. 画面右上の「共有」ボタンをタップ
2. 「CSVで出力」を選択
3. 保存先を選択（メール送信またはクラウド保存）
   推奨: メールで自分宛に送信
4. CSVファイルをダウンロード
```

---

### エクスポートされるCSVの形式

```csv
売上日,売上金額,客数,値引,取引件数
2025-11-25,"¥150,000",85,"¥0",80
2025-11-26,"¥180,000",92,"¥0",87
```

---

### 本システムへのインポート準備

#### 問題点

❌ **店舗IDが含まれていない**
❌ **仕入データが含まれていない**
❌ **通貨記号（¥）とカンマが含まれている**

#### 解決方法

**方法1: 店舗IDを手動追加（推奨）**

```csv
売上日,店舗ID,売上金額,客数
2025-11-25,YOUR_STORE_ID,"¥150,000",85
2025-11-26,YOUR_STORE_ID,"¥180,000",92
```

店舗IDの確認方法:
```
本システム → 店舗管理 → 該当店舗を選択
→ URLまたは詳細画面にIDが表示されます
```

**方法2: インポート時にデフォルト店舗を指定**

```
1. CSVはそのまま使用
2. インポート設定で「デフォルト店舗」を選択
3. インポート実行
```

---

### 完成例

#### テンプレートに合わせた形式

```csv
日付,店舗ID,売上,客数
2025-11-25,abc123-def456-ghi789,"150,000",85
2025-11-26,abc123-def456-ghi789,"180,000",92
```

---

### トラブルシューティング

#### Q: 通貨記号とカンマが問題になる？

**A:** いいえ、自動で処理されます。

```
"¥150,000" → 150000（自動変換）
```

#### Q: 取引件数データはどうなる？

**A:** 本システムでは客数として扱われます。

```
Airレジ「取引件数」→ 本システム「客数」
```

#### Q: 複数店舗のデータを一度にエクスポートできる？

**A:** Airレジでは店舗ごとにエクスポートが必要です。

```
店舗A → エクスポート → 店舗IDを追加
店舗B → エクスポート → 店舗IDを追加
→ 2つのCSVを別々にインポート
```

### CSVエクスポート手順

#### ステップ1: 管理画面にログイン

```
1. PCでスマレジ管理画面にアクセス
   https://www1.smaregi.jp/
2. ID・パスワードでログイン
```

#### ステップ2: 売上分析メニューを開く

```
1. 左メニューから「売上」をクリック
2. 「売上分析」を選択
3. 「日別売上」をクリック
```

#### ステップ3: 期間とフィルターを設定

```
1. 期間設定
   - 開始日: エクスポート開始日
   - 終了日: エクスポート終了日

2. 店舗フィルター（複数店舗の場合）
   ✅ 全店舗を選択: 全店舗データを一括出力
   ✅ 個別選択: 特定店舗のみ

3. 「表示」ボタンをクリック
```

#### ステップ4: CSVエクスポート

```
1. 画面右上の「CSV出力」ボタンをクリック
2. 出力項目を選択（全選択推奨）
   ✅ 日付
   ✅ 店舗名
   ✅ 売上
   ✅ 来店客数
   ✅ 原価
   ✅ 粗利
3. 「出力」ボタンをクリック
4. ダウンロード完了
```

---

### エクスポートされるCSVの形式

```csv
日付,店舗コード,店舗名,売上,来店客数,原価,粗利
2025-11-25,001,渋谷店,150000,85,45000,105000
2025-11-26,001,渋谷店,180000,92,52000,128000
```

---

### 本システムへのインポート準備

#### 確認事項

✅ **店舗コードが含まれている** → そのまま使用可
✅ **数値にカンマなし** → そのまま使用可
✅ **日付形式が正しい** → そのまま使用可

#### 店舗IDの置き換え（必要な場合）

スマレジの店舗コードを本システムの店舗IDに置き換えます：

**方法1: Excelで一括置換**

```
1. CSVをExcelで開く
2. Ctrl+H（置換）
3. 検索: 001（スマレジの店舗コード）
4. 置換: abc123-def456-ghi789（本システムの店舗ID）
5. 「すべて置換」をクリック
6. CSV UTF-8形式で保存
```

**方法2: デフォルト店舗を使用**

```
単一店舗の場合:
→ 店舗コード列を削除
→ インポート時にデフォルト店舗を指定
```

---

### 完成例

```csv
日付,店舗ID,売上,客数,仕入
2025-11-25,abc123-def456-ghi789,150000,85,45000
2025-11-26,abc123-def456-ghi789,180000,92,52000
```

---

### トラブルシューティング

#### Q: 店舗名が日本語だけどOK？

**A:** はい、問題ありません。

```
本システムは店舗IDで識別するため、
店舗名列は無視されます。
```

#### Q: 粗利データはインポートされる？

**A:** いいえ、自動計算されます。

```
本システム:
粗利 = 売上 - 仕入 - 経費
→ 自動計算されるため入力不要
```

#### Q: 時間別売上もエクスポートできる？

**A:** 本システムでは日次単位のみ対応です。

```
時間別データ → 日次に集計してインポート
```

#### Q: 客単価はインポートされる？

→ 本システムで自動計算

```
商品別データ → 合計して日次データに
```

#### Q: 時間帯別データはどうする？

**A:** ランチ・ディナーの場合は営業区分で対応。

```
ランチタイム → operation_type: lunch

```

---

### エクスポートされるCSVの形式

```csv
Date,Location,Gross Sales,Refunds,Net Sales,Discounts,Number of Transactions,Average Sale
2025-11-25,Tokyo Store,$1500.00,$0.00,$1500.00,$0.00,85,$17.65
2025-11-26,Tokyo Store,$1800.00,$0.00,$1800.00,$0.00,92,$19.57
```

---

### 本システムへのインポート準備

#### 確認事項

⚠️ **通貨がドル表記** → 日本円に変換が必要
⚠️ **カラム名が英語** → 自動マッピング対応
✅ **Date形式が標準** → そのまま使用可

#### 通貨変換

**Excelで一括変換:**

```
1. CSVをExcelで開く
2. Gross Sales列を選択
3. 数式を適用:
   = VALUE(SUBSTITUTE(A2,"$","")) * 150
   （$1 = ¥150 として計算）
4. 通貨記号を削除
5. CSV UTF-8形式で保存
```

#### カラム名の対応

| Square | 本システム |
|--------|-----------|
| Date | 日付 |
| Gross Sales | 売上 |
| Number of Transactions | 客数 |
| - | 仕入（データなし） |

---

### 完成例

```csv
Date,店舗ID,Gross Sales,Number of Transactions
2025-11-25,abc123-def456-ghi789,150000,85
2025-11-26,abc123-def456-ghi789,180000,92
```

---

### トラブルシューティング

#### Q: 米ドル表記はそのままでOK？

**A:** いいえ、日本円に変換してください。

```
方法1: レート計算（推奨）
  $100.00 × 150 = ¥15,000

方法2: Squareの設定で通貨を変更
  Settings → Regional → Currency → JPY
```

#### Q: Refunds（返金）はどう扱う？

**A:** Net Salesを使用してください。

```
Net Sales = Gross Sales - Refunds
→ 実質の売上として扱われます
```

#### Q: Tips（チップ）は含まれる？

**A:** はい、Gross Salesに含まれています。

```
日本の飲食店ではチップ文化がないため、
そのまま売上として計上してOK
```

## その他のPOSレジ

### 汎用的な対応方法

本システムのテンプレートに合わせて整形してください。

---

### ステップ1: CSVエクスポート

各POSレジの「売上レポート」や「日次データ」をCSV形式でエクスポートします。

一般的な手順：
```
1. 管理画面/バックオフィスにログイン
2. レポート/集計メニューを開く
3. 日別売上または日次レポートを選択
4. 期間を指定
5. CSVエクスポート
```

---

### ステップ2: テンプレートをダウンロード

```
1. 本システムにログイン
2. データ管理 → CSVインポート
3. 「テンプレートダウンロード」をクリック
4. 日次レポートテンプレートを取得
```

---

### ステップ3: データを整形

#### Excelで整形する場合

```
1. POSからエクスポートしたCSVを開く
2. テンプレートを別シートで開く
3. カラム名を確認
4. データをコピー&ペースト
   - 日付 → report_date列
   - 売上 → sales列
   - 客数 → customers列
   - 仕入 → purchases列
5. 店舗IDを追加（全行に同じIDを入力）
6. CSV UTF-8形式で保存
```

#### Google スプレッドシートで整形する場合

```
1. Google Driveでスプレッドシート作成
2. ファイル → インポート → CSVアップロード
3. テンプレートの形式に合わせて編集
4. ファイル → ダウンロード →
   カンマ区切りの値（.csv、現在のシート）
```

### よくある問題と解決方法

#### 問題1: 日付形式が違う

```
× 不正な形式:
  11/25/2025
  2025年11月25日
  Nov 25, 2025

✅ 正しい形式:
  2025-11-25

Excelでの変換:
  = TEXT(A2, "yyyy-mm-dd")
```

#### 問題2: 数値にカンマが含まれる

```
× 問題: 150,000 → 分割される

✅ 解決方法1: 引用符で囲む
  "150,000"

✅ 解決方法2: カンマを削除
  150000
```

#### 問題3: 店舗IDがわからない

```
確認方法:
  1. 本システムにログイン
  2. 店舗管理ページを開く
  3. 該当店舗をクリック
  4. URL または詳細画面にIDが表示

例: https://example.com/stores/abc123-def456-ghi789
                                  ↑
                               これが店舗ID
```

#### 問題4: 複数店舗のデータ

```
方法1: CSV内に店舗IDを記載
  日付,店舗ID,売上
  2025-11-25,店舗A_ID,150000
  2025-11-25,店舗B_ID,180000

方法2: 店舗ごとに分けてインポート
  店舗A.csv → インポート（デフォルト: 店舗A）
  店舗B.csv → インポート（デフォルト: 店舗B）
```

## 共通のベストプラクティス

### データ品質の確保

#### チェックリスト

```
□ 日付が正しい形式（YYYY-MM-DD）
□ 店舗IDが正しい（本システムのID）
□ 数値に余計な文字が含まれていない
□ ヘッダー行が存在する
□ 空行が含まれていない
□ 文字コードがUTF-8
```

### 定期的なインポート

#### 推奨スケジュール

```
毎日: 日次レポート
  営業終了後（22:00-23:00）
  → POSで締め処理
  → CSVエクスポート
  → インポート（2分）

毎月: 月次経費
  月末または月初
  → 経費データを整理
  → CSVインポート
```

### バックアップ

```
推奨:
  1. エクスポートしたCSVを保管（クラウド推奨）
  2. インポート前にバックアップを確認
  3. エラーが発生した場合に備える

保管場所:
  - Google Drive
  - Dropbox
  - OneDrive
  - 社内サーバー
```
